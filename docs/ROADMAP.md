# 斩币 — 产品路线图

> 独立开发者 · Web 原型验证优先 · 国内先行出海 · 混合变现

---

## 1. 产品定位

- **目标用户**：休闲+中度玩家，喜欢"上手简单、精通困难"的游戏
- **核心卖点**：切水果的爽感 × 合成大西瓜的满足感 × Roguelike 的成长深度
- **差异化**：划砍不只是输入方式，更是经济系统的操控手段（刀光拨币制造合成）
- **一局时长**：5-10 分钟，适合碎片化时间

---

## 2. 阶段路线

### Phase 0：Web 原型验证（第 1 个月）

**目标：** 验证核心手感——划砍 + 产物物理堆叠 + 合成是否好玩

**只做这些：**
- 上半屏：基础敌人直落 + 划砍碰撞检测 + 切中反馈
- 下半屏：产物物理掉落堆叠 + 3 合 1 基础合成（无分支）
- 刀光拨币交互
- 基础失败条件（产物被吃光）
- **不做**：遗物、成就、局外养成、分支合成树

**验证方式：**
- 找 10-20 人试玩，观察是否自发重复玩
- 核心问题：划砍有没有爽感？合成有没有满足感？
- 如果手感不对，在这个阶段调参数，不往下走

**交付物：** 可在手机浏览器打开的 HTML5 页面

---

### Phase 1：最小完整版（第 2-3 个月）

**目标：** 一局完整的游戏体验

**在原型基础上新增：**
- 波次系统（敌人密度/速度递增）
- 侧抛型敌人 + 炸弹型敌人
- 分支合成树（2-3 条分支）
- 增益系统（主动消耗 + 稀有度里程碑赠送）
- 遗物系统（基础 8-10 个遗物）
- 完善的失败 + 结算画面
- 基础 UI 和音效

**决策点：** 根据反馈选择下一步平台：
- 留存好 → 微信小游戏上线
- 操作体验好 → 移动端 App 开发
- 都一般 → 回头调整核心玩法

---

### Phase 2：小游戏上线（第 4-5 个月）

**目标：** 微信/抖音小游戏首发，验证市场

**新增内容：**
- 局外养成（遗产系统、内容解锁、永久微提升）
- 成就系统（首批 30-40 个成就）
- 图鉴系统
- 排行榜（好友 + 全服）
- 分享功能（成就截图、最高波次）

**变现植入：**
- 激励视频广告：死亡后看广告续命一次 / 遗产翻倍
- 插屏广告：每 3 局结算后
- 暂不做内购，先跑数据

**关键指标：**
- 次日留存 > 40%（健康）
- 7 日留存 > 15%（有潜力）
- 单用户广告 ARPU
- 自然分享率

---

### Phase 3：移动端 App 上线（第 6-8 个月）

**前提：** 小游戏数据验证 OK

**新增内容：**
- 完整合成分支树（所有流派展开）
- 更多遗物、增益、敌人种类
- 赛季系统（每赛季重置排行榜 + 新遗物/增益入池）
- 内购商城上线

**目标平台：** iOS App Store + Android（国内各渠道 + Google Play）

---

### Phase 4：内容扩展 + 出海（第 9 个月起，持续）

**长线运营：**
- 本地化（英语、日语优先）
- Google Play / App Store 海外版
- Steam 移植（鼠标划砍适配）
- 持续赛季更新：每赛季新遗物 + 新合成分支 + 新敌人

---

## 3. 变现设计

### 3.1 广告（Phase 2 起）

| 广告类型 | 场景 | 频率 |
|---|---|---|
| 激励视频 | 死亡后续命 | 每局最多 1 次 |
| 激励视频 | 结算时遗产翻倍 | 每局最多 1 次 |
| 插屏广告 | 每 3 局结算后 | 可控频率 |

### 3.2 内购（Phase 3 起）

**皮肤包（纯外观，不影响数值）**
- 刀光特效皮肤（墨剑、星刃、金焰等）
- 产物外观皮肤（不同主题的金币/宝石外观）
- 背景主题

**赛季通行证**
- 免费轨道：遗产 + 基础皮肤
- 付费轨道：高级皮肤 + 称号框 + 专属特效

**局外养成加速**
- 用付费货币加速解锁内容
- 只卖时间，不卖数值

### 3.3 红线（绝对不卖）

- 遗物
- 增益
- 永久属性提升
- 任何影响局内公平性的东西

---

## 4. 社交与传播策略

### 4.1 天然想分享的时刻

- 首次合出高级产物（截图带产物发光效果）
- 达成稀有成就/隐藏成就
- 出现超长连锁合成
- 创纪录最高波次

### 4.2 传播机制

- 成就截图自动带游戏水印/小程序码
- 排行榜挑战好友
- 隐藏成就激发社区讨论（"那个 ??? 怎么触发？"）
- 产物组合截图分享（"我同时凑齐了三系最高级！"）

### 4.3 社区运营

- 建立核心玩家群，收集反馈
- 定期在社媒发布"本周最稀有产物组合"
- 隐藏成就攻略引导 UGC 内容产出

---

## 5. 版号与合规

- 小游戏渠道对版号要求相对宽松，Phase 2 可先上线
- Phase 1 完成时启动版号申请材料准备（流程 3-6 个月）
- 版号申请中可同步：国内继续跑小游戏 + 海外先上架
- 关注内容合规：暴力程度、付费设计等审核要点

---

## 6. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 核心手感不达标 | 致命 | Phase 0 快速验证，不过关不往下走 |
| 物理引擎性能瓶颈 | 高 | Phase 0 压测 100+ 物体，提前评估替代方案 |
| 合成碎片化 | 中 | 控制分支数量，前期集中后期展开 |
| 独立开发精力分散 | 中 | 严格按阶段做，每阶段只做该阶段的事 |
| 版号申请周期长 | 中 | Phase 1 就启动申请，小游戏+海外过渡 |
| 变现 ARPU 偏低 | 中 | 先跑数据，Phase 3 补充内购 |

---

## 7. 里程碑时间线

```
月份     阶段              关键产出                        决策点
─────────────────────────────────────────────────────────────────
 1       Phase 0          Web 可玩原型                    手感验证
 2-3     Phase 1          完整单局体验                    平台选择
 4-5     Phase 2          微信小游戏上线                  数据验证
 6-8     Phase 3          移动端 App 开发                 —
 9       Phase 3 上线      App Store / Google Play         —
 10+     Phase 4          出海 + 长线运营                  —
```

每个阶段结束都有明确的决策点，避免在错误方向上投入过多。
